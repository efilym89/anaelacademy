# Checklist для нового проекта

## 1) Базовая настройка

- [ ] Определить стек (frontend/backend/fullstack).
- [ ] Выбрать менеджер пакетов и зафиксировать версию.
- [ ] Настроить CI (lint + test + build).
- [ ] Настроить линтеры/форматтеры для выбранного стека.

## 2) Инфраструктура

- [ ] Описать зависимости в Docker (если нужен контейнерный запуск).
- [ ] Подготовить dev/prod конфигурации.
- [ ] Настроить хранение секретов (не в репозитории).

## 3) Качество и безопасность

- [ ] Добавить минимальные тесты (smoke test).
- [ ] Подключить проверку уязвимостей зависимостей.
- [ ] Добавить правила ветвления и code review.

## 4) Документация

- [ ] Уточнить целевую архитектуру проекта.
- [ ] Добавить раздел "Как запускать локально".
- [ ] Описать процесс релиза.
